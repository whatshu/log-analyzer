# log-analyzer

## design

该项目是一个长文本日志分析器, 前端 cli 基于 python 实现, 后端的执行代码需要使用 rust 以及高性能多线程设计.

其具有如下实体.

1. 日志仓, 保存了日志原文, 以及一些元信息, 保存的时候可以使用压缩. 日志仓可以复制, 并记录操作用于回退等. 你可以参考 git 的实现.
2. 操作符, 将被应用于某一个日志仓. 所有操作符都应该可以被反向操作, 用于撤回操作.

其具有如下功能.

1. 从纯文本导入日志仓并保存, 导入的过程中会自动创建元数据.
2. 创建操作符, 使用格式化的语句实现复杂功能, 至少需要支持正则表达式.
3. 操作符的输入和输出均为文本, 日志文本将被按行执行.
4. 确保实现每行文本的增删改查.

同时, 该项目需要针对 ai agent 进行优化, 提供合适的 skill 以及详细的描述.

使用结构化的 python + rust 项目设计, 不要把所有代码都写到一起.

该项目将被用于处理 >10GB 的文本日志, 所以你需要关心性能问题.

你需要创建合适的 rust 后端, 以及 python 前端, 添加合适且完备的单元测试以及场景测试, 你可以模拟一些日志文件进行真实的调用测试.

## 优化

参考 klogg 等长文本分析展示软件, 提升日志操作效率.

记住, 多参考现有的高性能文本处理软件设计, 在可能的时候直接使用他们作为该项目的依赖.
