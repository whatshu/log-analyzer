# Performance Benchmarks

> Generated: 2026-04-17  
> CPU: AMD Ryzen 9 9955HX 16-Core (5.0 GHz)  
> RAM: 60 GB  
> Storage: NVMe SSD  
> OS: Linux 6.18 (Fedora 43)  
> Python: 3.14.3  

## Test File

Synthetic log file generated by `scripts/bench.py`:

| Property | Value |
|----------|-------|
| Size | **10.00 GiB** |
| Lines | **129,249,280** |
| Format | `YYYY-MM-DD HH:MM:SS [LEVEL] tid=NNN <message>` |
| Level mix | 5% ERROR / 10% WARN / 60% INFO / 25% DEBUG |
| Avg line length | ~150 bytes |
| Pattern searched | `ERROR` (~5% of lines = 6,103,040 matches) |

All benchmark times are the **median of 3 consecutive runs** (warm OS page cache).
Throughput is calculated relative to the 10 GiB raw file.

---

## 1. Line counting

| Tool | Time | Throughput |
|------|------|------------|
| `wc -l` | **0.79 s** | 12.6 GiB/s |
| `rg -c '.'` | 5.25 s | 1.9 GiB/s |

`wc -l` wins by scanning only for `\n` bytes without invoking a regex engine.
`rg -c '.'` applies its regex engine per line, slower for this specific task.

---

## 2. Pattern search — count lines matching `ERROR`

| Tool | Time | Throughput | Notes |
|------|------|------------|-------|
| `awk '/ERROR/{c++}END{print c}'` | 36.23 s | 0.28 GiB/s | pure interpreter, no SIMD |
| `grep -c ERROR` | 1.95 s | 5.1 GiB/s | single-threaded, SIMD |
| `log-analyzer count_file_matches()` | 1.53 s | 6.5 GiB/s | ripgrep SIMD on raw file, no import |
| `rg -c ERROR` | 1.18 s | 8.5 GiB/s | multi-threaded, SIMD |
| **`log-analyzer count_matches()` (compressed)** | **0.51 s** | **19.6 GiB/s*** | reads only 4.26 GiB compressed |

\* Throughput marked `*` is normalized to raw-equivalent bytes.
  Actual compressed I/O is 4.26 GiB.

**Key result:** after the one-time import, `count_matches()` on the compressed
repo is **2.3× faster than `rg`** on the raw file, because it reads 2.4× less data.

---

## 3. Import & compression (one-time cost)

| Metric | Value |
|--------|-------|
| Import time | 11.7 s |
| Import throughput | 0.86 GiB/s |
| Raw file | 10.00 GiB |
| Compressed repo | 4.26 GiB |
| **Compression ratio** | **2.4×** |

The import reads the raw file once, builds a line index, and writes
zstd-compressed chunks. All subsequent operations work on the smaller repo.

---

## 4. Filter matching lines → output file

| Tool | Time | Throughput | Notes |
|------|------|------------|-------|
| `grep ERROR file > out` | 2.23 s | 4.5 GiB/s | single-threaded |
| `rg ERROR file > out` | **1.42 s** | **7.1 GiB/s** | multi-threaded |
| `log-analyzer stream_filter_to_file()` | 9.21 s | 1.1 GiB/s | decompress + scan + write |

For write-heavy filter operations, `grep`/`rg` are faster. `log-analyzer`'s
streaming filter pays both a decompression cost and a per-line write overhead.
This is the primary area for future native optimization.

---

## Summary

| Scenario | Best tool | Relative speed |
|----------|-----------|----------------|
| One-time line count | `wc -l` | — |
| **Repeated pattern counting** | log-analyzer (compressed) | **2.3× faster than `rg`** |
| Filter to file | `rg` | — |
| Reversible ops (filter/replace/undo) | log-analyzer | unique feature |
| Disk footprint | log-analyzer repo | **2.4× smaller** |

### When log-analyzer pays off

- **Repeated search / count queries** on the same dataset: the 2.4× smaller
  compressed repo means each query reads proportionally less I/O.
- **Mixed workloads** (filter → inspect → undo → re-filter): the operation
  journal lets you iterate without re-reading the raw 10 GB file.
- **Python API integration**: programmatic access, no subprocess spawning.

### When raw tools win

- **Single-pass filter to file**: `grep` / `rg` have optimized pipelines for this.
- **One-off queries**: if you query only once, the 11.7 s import overhead
  is not amortized.
- **Simple line counting**: `wc -l` is always fastest for line counts.

## Reproducing

```bash
# Generate a fresh 10 GiB file, run benchmarks, then clean up automatically
python3 scripts/bench.py

# Smaller file for a quick test
python3 scripts/bench.py --gb 2

# Benchmark an existing file (skips generation)
python3 scripts/bench.py --file /path/to/your.log
```
